Access Review Details - Filter
The Access Review Details filter enables you to control the number of items displayed in the access review list. You can use any combination of filters.
To define a filter, click the + icon, specify filter options and then click Add Filter. To update the access review list, click Filter. Click Reset to remove the filter and view all items included in this access review request.
The following filters are available:
 
Table 1—Access Review Details - Filters

Status
Note: Some status options are dependent on IdentityIQ configuration settings and may not be available in all access reviews.
Filter by access review status.
Only one status can be specified per filter.
Open — action is required on this item before this access review request is considered complete.
Complete — access review of this item is complete. Requesting revocation moves the state to complete as well.
Delegated — access review for the items were delegated to another approver.
— OR —
One or more entitlement or policy violations belonging to the identity was delegated to another approver.
That approver has not yet taken action on the delegated access review request.
Waiting Review — action was taken on a delegated access review request and that action is now awaiting your review.
Returned — the access review request for this item was delegated and returned with no action.
Challenged — the affected user challenged a revocation request and that challenge must be acted upon before the item is complete.
Changes Detected
Note: This option is not available on account group or role access review types.
Yes — show only identities with identity or entitlement information that has changed since the last access review.
No — show only identities with identity and entitlement information that has not changed since the last access review.
Add Filter
Note: Some options are dependent on the access review type and IdentityIQ configuration settings and may not be available in all access reviews.
Display additional filter options. Unless you specify multiple filters of the same type, the filters are inclusive, only items matching all of the criteria specified are displayed in the list. For example, if you filter by First Name John and Last Name Doe, only identities with the name John Doe are included in the list.
If you specify multiple filters of the same type, First Name John and First Name Joe. Identical filter types are controlled with an OR statement and both values are returned.
Select a filter from the Filter by: drop-down list and a value to associate with that filter from the drop-down list. The value field contains all the valid options for the filter selected.

Access Review Properties:
Additional Entitlements Detected — select yes or no to return all identities that have or do not have additional entitlements.
Continuous State — items in the state specified, certified, access review required, or overdue.
Item Type — items of the type selected, for example policy violations or additional entitlements.
Next Continuous Change — returns items that change state on, before, or after the specified date.
Role — all identities that have the entitlements that make up the Role specified.
Policy Violations Detected — select yes or no to return all identities that have or do not have policy violation detected.

Identity Properties:
You can filter on identity properties including first and last name, location, and organization. When you select Risk Score and provide a value, all identities with the minimum risk score specified are returned

Account Properties:
Account properties are defined during system configuration and can be any account attribute from any application configured to work with IdentityIQ.

Additional Entitlements:
The Additional Entitlements list contains all of the applications containing additional entitlements assigned to identities in the access review. Specify an application and use the drop-down list to select the specific entitlement and value of interest.
